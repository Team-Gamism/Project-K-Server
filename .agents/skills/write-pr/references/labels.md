# GitHub Labels Reference

PR에 할당할 라벨을 **1~2개** 선택합니다.

## PR 할당 가능 라벨

| 라벨 | 사용 시점 |
| --- | --- |
| `enhancement:개선작업` | 새 기능 추가, 기존 기능 개선, 리팩토링 |
| `bug:버그` | 버그 수정 |
| `documentation:문서화` | README, CODEX, 주석 등 문서만 변경 |
| `release:릴리즈` | 릴리즈 준비 또는 버전 업 |

## 할당 금지 라벨 (수동 전용)

| 라벨 | 이유 |
| --- | --- |
| `waiting for review:검토 대기` | PR 작성자가 수동 적용 |
| `blocked:차단됨` | 수동으로만 적용 |
| `help wanted:도움 필요` | 이슈 전용 |
| `duplicate:중복` | 이슈 전용 |
| `invalid:무효한` | 이슈 전용 |

## 빠른 결정

```
버그 수정?         → bug:버그
새 기능 / 개선?    → enhancement:개선작업
문서만 변경?       → documentation:문서화
릴리즈?            → release:릴리즈
모르겠음?          → enhancement:개선작업
```
